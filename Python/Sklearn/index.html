    voting¶

    投票分类器每个预测器使用不同的算法

    硬投票和软投票：硬投票选择投票数最多的类，软投票平均每个分类器的类别后给出预测，通常效果更好，因为会给予高度自信的投票更高的权重（voting="hard" && voting="soft"）

    bagging 和 pasting¶

    每个预测器使用相同的算法；可以通过不同的 CPU 内核甚至不同的服务器并行的训练和预测，非常易于扩展

    bagging：自举汇聚法，采样时有放回，pasting：采样时无放回；换句话说，两者都允许一个样本被多次采样，但只有 bagging 允许一个样本被一个预测器多次采样

    bagging 因为自举法给训练子集引入了更高的多样性所以偏差比 pasting 略高，但也意味着预测器之间的关联度更低，所以集成的方差小

    包外评估：

    clf = BaggingClassifier(DecisionTreeClassifier(), n_estimators=500, bootstrap=True, obb_score=True)
clf.fit(x_train, y_train)
print(clf.oob_score_)               # 包外评估分数
print(clf.oob_decision_function_)   # 包外决策函数，返回概率

    随即补丁和随机子空间¶

    随机补丁：对训练样本和特征都进行抽样

    随机子空间：保留所有训练实例（bootstrap=False, max_samples=1.0），但对特征进行抽样（bootstrap_features=True, max_features<1.0）

    随机森林¶

    随机森林是决策树的集成

    极端随机树（ExtraTrees）：

    • 随机森林在单棵树每个结点分裂时对每个特征使用随机阈值，而不是搜索得出最佳阈值

    • 以更高的偏差换取了更低的方差，且训练起来快得多（因为寻找最佳阈值是决策树生长中最耗时的任务）

    特征重要性：

    随机森林可以计算特征重要性；通过查看所有树上使用该特征的树结点 加权平均 减少不纯度的程度来衡量重要性，权重为其关联的训练样本的数量

    boosting¶

    提升法；缺点是无法并行，所以在扩展方面不如 bagging 和 pasting；

    AdaBoost：对前面的分类器分类错的样本加大权重再训练一个新的分类器

    梯度提升：

    • 让新的预测器针对前一个预测器的残差进行拟合，然后结果相加；比如 GBRT（梯度提升回归树）
    • 超参数 learning_rate 对每棵树的贡献进行缩放，如果是低值则需要更多的树来拟合，但预测的泛化效果通常更好
    • 要找到树的最佳数量可以使用提前停止法 (staged_predict() 或者 warm_start())
    # staged_predict() 
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_squared_error
X_train, X_val, y_train, y_val = train_test_split(X, y)
gbrt = GradientBoostingRegressor(max_depth=2, n_estimators=120)
gbrt.fit(X_train, y_train)
errors = [mean_squared_error(y_val, y_pred)
          for y_pred in gbrt.staged_predict(X_val)]
bst_n_estimators = np.argmin(errors) + 1
# 重新训练一个
gbrt_best = GradientBoostingRegressor(max_depth=2,n_estimators=bst_n_estimators)
gbrt_best.fit(X_train, y_train)

# warm_start()
gbrt = GradientBoostingRegressor(max_depth=2, warm_start=True)
   min_val_error = float("inf")
   error_going_up = 0
   for n_estimators in range(1, 120):
       gbrt.n_estimators = n_estimators
       gbrt.fit(X_train, y_train)
       y_pred = gbrt.predict(X_val)
       val_error = mean_squared_error(y_val, y_pred)
       if val_error < min_val_error:
           min_val_error = val_error
           error_going_up = 0
       else:
           error_going_up += 1
           if error_going_up == 5:  # 验证集误差连续5此未改善
               break  # early stopping
    • XGBoost （Extreme Gradient Boosting）提供了梯度提升的优化实现，且 API 和 sklearn 相似

    stacking¶

    堆叠法

    将训练集分成 n 部分，第 1 部分训练第一层的 m 个预测器，然后把第 2 部分的样本输入第一层的预测器得到新的样本来训练第二层的预测器 ........ 最上层用一个预测器来得到最终的结果

    sklearn 不支持堆叠，需要自己手动实现 或者 使用开源的方案，如 DESlib

    Pipeline¶

    class sklearn.pipeline.Pipeline(steps, *, memory=None, verbose=False)

    Pipeline of transforms with a final estimator. 最后一个是估算器前面都是转换器（也就是说，必须要有 fit_transform() 方法）；当调用流水线的 fit() 方法时依次调用转换器的 fit_transform() 然后调用最后的估算器的 fit()
