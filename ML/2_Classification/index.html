    Classification¶

    概述¶

    分类，function 的结果是离散型的，必然属于结果集中的某一个，比如宝可梦的属性（当然宝可梦有多属性）

    Regression？¶

    Regression 并不适用于 output 是离散的函数

    而且需要注意的是如果是 多元分类，不能把 class1's target 设为 1， class2's target 设为 2，class3's target 设为 3，因为这样做就默认 class1 和 class2 相比 和 class3 更加接近

    Ideal Alternatives¶

    Function Model：function f(x) 里面设另外一个function g(x)

    $$ \begin{align} g(x)>0\ \ output=class1 \ g(x)<0\ \ output=class2 \end{align} $$ 为什么又能用数值表示 class 了？ 因为那是针对 Regression 的 Loss function 而言的，所以我们需要重新定义一个 Loss Function $$ Loss=\sum\limits_n\delta(f(x^n)≠\hat{y}^n) $$ 不过这个 Loss Function 无法微分，也有 Perceptron、SVM 这些方法可以用，但这里先用另外一个solution来解决这个问题

    Generative model¶

    基于概率论中的 贝叶斯公式：\(P(A|B)=\frac{P(AB)}{P(B)}\)

    在已知 input x 的情况下计算 \(P(C_1|x)\) 和 \(P(C_2|x)\) $$ P(C_1|x)=\frac{P(C_1)P(x|C_1)}{P(C_1)P(x|C_1 )+P(C_2)P(x|C_2)} $$

    \[ P(C_2|x)=\frac{P(C_2)P(x|C_2)}{P(C_1)P(x|C_1)+P(C_2)P(x|C_2)} \]

    Prior¶

    模型需要得到四个值 \(P(C_1) \ P(C_2) \ P(x|C_1) \ P(x|C_2)\)

    \(P(C_1) \ P(C_2)\) ：

    算出 training data 中 属于 class1 和 class2 各自占总样本量的比例即可

    \(P(x|C_1) \ P(x|C_2)\)：

    多维高斯分布：期望 \(u\) 和 协方差 \(\Sigma\) 都是 matrix $$ f_{u,\Sigma}(x)=\frac{1}{(2\pi)^{\frac{D}{2}}}\frac{1}{|\Sigma|^{\frac{1}{2}}}exp{-\frac{1}{2}(x-u)^T\Sigma^{-1}(x-u)} $$ 利用极大似然估计 \(u^*,\Sigma^*=\arg \max\limits_{u,\Sigma} L(u,\Sigma)\) 去估计 \(u，\Sigma\) 的取值，这里的极大似然估计的值就是评价了这组参数的好坏，我们要找到使极大似然估计值最大的一组参数，其实就是所有样本点的均值和协方差

    注：数学期望：\(u=E(X)\)，协方差：\(\Sigma=cov(X,Y)=E[(X-u)(Y-u)^T]\)，对同一个变量来说，协方差为 \(cov(X,X)=E[(X-u)(X-u)^T]=D(X)\)

    得到 \(x|C_1 \ x|C_2\) 的高斯分布后把 \(x\) 代入即可得到 \(P(x|C_1) \ P(x|C_2)\)

    Do Classification¶

    \[ \begin{align} if \ P(C_1|x) > 0.5 \Rightarrow x \ belongs \ class1 \\ if \ P(C_2|x) > 0.5 \Rightarrow x \ belongs \ class2 \end{align} \]

    在只考虑宝可梦的 Defense 和 SP Denfense 时这样做出来的效果的出来的准确率并不高，即使考虑了6个 features 准确率也只有 64%，表现比较糟糕

    Modifying Model¶

    上面的 model 其实并不常见，因为一般不会每个 Gaussion 都有自己的 mean 和 covariance，比如我们的 class1 用的是 \(u_1\) 和 \(\Sigma_1\)，class2 用的是 \(u_2\) 和 \(\Sigma_2\)，比较常见的做法是，不同的 class 可以share 同一个 cocovariance matrix

    其实 variance 是跟 input 的 feature size 的平方成正比的，所以当 feature 的数量很大的时候，\(\Sigma\) 大小的增长是可以非常快的，在这种情况下，给不同的 Gaussian 以不同的 covariance matrix，会造成model 的参数太多，而参数多会导致该 model 的 variance 过大，出现 overfitting 的现象，因此对不同的 class 使用同一个 covariance matrix，可以有效减少参数

    此时就把 \(u_1\)、\(u_2\) 和共同的 \(\Sigma\) 一起去合成一个极大似然函数，此时会发现，得到的 \(u_1\) 和 \(u_2\) 和原来一样，还是各自的均值，而 \(\Sigma\) 则是原先两个 \(\Sigma_1\) 和 \(\Sigma_2\) 的加权

    结果可视化后会发现考虑 2 个参数时，分界线由曲线变成了直线

    这样的 model 也称之为 linear model (尽管 Gaussian 不是 linear 的，但是它分两个 class 的boundary 是 linear)

    如果再把考虑所有的 feature，那么准确率将会变成 74%，但高维空间无法可视化

    Naive Bayes Classifier¶

    朴素贝叶斯分类法

    为了减少参数假设 \(x=[x_1,x_2,...x_n]\) 每个参数之间是相互独立的，那每一个参数都可以写一个一维的 Gaussion，\(\Sigma\) 除了对角线之外都是 0，这样得出的结果会比较糟糕，因为显然战斗力和防御力是成正比的，covariance 不可能为0

    总之，寻找 model 总的原则是，尽量减少不必要的参数，但是必然的参数绝对不能少

    那怎么去选择分布函数呢？有很多时候凭直觉就可以看出来，比如宝可梦有某个feature是binary的，它代表的是：是或不是，这个时候就不太可能是高斯分布了，而很有可能是伯努利分布(两点分布)

    Analysis Posterior Probability¶

    分析一下后验概率 \(P(C_1|x)\)

    表达式同除以分子，可以得到 \(\sigma(z)=\frac{1}{1+e^{-z}}\)，这个function叫做sigmoid function（S函数）

    推导一下 Z 的真正的样子

    推导过程比较复杂，但是结果很简单： $$ z=(\mu^1-\mu^2)^T\Sigma^{-1}x-\frac{1}{2}(\mu^1)^T\Sigma^{-1}\mu^1+\frac{1}{2}(\mu^2)^T\Sigma^{-1}\mu^2+ln\frac{N_1}{N_2} $$ 可以看成：\(z=w^Tx+b\)

    当 \(\Sigma_1 \ \Sigma_2\) 共用一个 \(\Sigma\) 并经过化简相消后，\(x\) 的系数 \(w\) 是一个 vector，后面的是一个常数项 b

    \(P(C_1|x)=\sigma (w\cdot x+b)\) 这个式子就解释了，当 class1 和 class2 共用\(\Sigma\) 的时候，它们之间的 boundary 会是 linear 的

    那么能不能直接求出 \(w，b\) 呢？
