    Convolutional Neural Network¶

    CNN 常常被用在影像处理上，它的 theory base 就是三个property，和两个架构 convolution 架构：针对property 1和property 2 max pooling 架构：针对property 3

    Why CNN for Image？¶

    CNN vs DNN¶

    在 train neural network 的时候，每一个 neuron 都代表了一个最基本的 classifier

    举例子来说，第一层 layer 做的事情就是 detect 有没有绿色， 黄色，斜条纹；然后第二层做更加复杂的事情，比如看到直线横线就是框窗框的一部分，看到棕色直条纹就是木纹；第三层 layer 更复杂...

    现在的问题是，如果我们直接用一般的 fully connected 的 feedforward network 来做图像处理的时候，往往会需要太多的参数

    一张很小的 100*100 的彩图需要 100*100*3 的 vector，加上 neuron 参数过多

    CNN 做的就是通过一些对影像处理的理解把一些用不到的参数过滤掉

    Three Property for CNN theory base¶

    能用较少的的参数来做图像处理的原因是以下三个对影像处理的观察，也是 CNN 架构提出的基础

    Some patterns are much smaller than the whole image¶

    在影像处理里，一些 neuron 做的事情是 detect 有没有一种东西或者 pattern 的出现，这个 pattern 是比整张 img 小很多的，所以不需要看整张图片；比如要判断图片有没有一张鸟的图，一些 neuron 看有没有鸟嘴，一些看有没有爪子，这些 neuron 都只需要连接到 img 上的一小块区域，不需要完整的图，所以对应着更小的参数

    The same patterns appear in different regions¶

    同样的 pattern 可能出现在 img 中不同的部分，比如鸟嘴可能在 img 中间，也可能在左上角，所以可以 share 同样的 neuron，同样的参数去 detect，这样能减小参数

    Subsampling the pixels will not change the object¶

    对一张 image 做 subsampling (二次抽样)，比如把奇数行，偶数列的 pixel 删去，并不对影响人对一张图片的理解，依旧可以识别图片中出现了什么，所以做 subsampling 也能减小参数

    The whole CNN structure¶

    CNN 的整体架构：通过 n 次的 Convolution + Max Pooling，这个 n 是在定这个network 的架构时就要事先决定好，然后做 Flatten，然后是一般的 Fully connected network 里面去，最终得到影像辨识的结果

    Convolution 的 layer 处理上面提到的前 2 个 Property，Max Pooling 处理第三个

    Convolution¶

    Property1¶

    每个 Filter 是一个 matrix，其中的每一个值都是 train 出来的，而不是人去设计

    每一个 Filter 的大小意味着他只会在这个区域内 detect

    Property2¶

    Filter 从左上角开始做 silde window，每次移动一定的距离，叫做 stride，也是自己设定，每次停下都做卷积，如下图所示，最终得到了一个 \(4\times4\) 的 matrix，其中的最大值 2 个 3 代表 detect 的 pattern 在左上和左下

    Feature Map：一个 convolution 的 layer 里面会有很多 filter，不同的 filter 的参数不同，每个 filter 都会得到一个 matrix，所有的 matrix 合起来就叫做 Feature Map(特征映射)，由多少个 filter，就有多少个映射后的 matrix

    CNN 对不同 scale 的相同 pattern 的处理上存在一定的困难，如果你今天有同一个 pattern，它有不同的 size，有大的鸟嘴，也有小的鸟嘴，CNN 并不能够自动处理这个问题；DeepMind 曾经发过一篇 paper，上面提到了当你 input 一张 image 的时候，它在 CNN 前面，再接另外一个 network，这个 network 做的事情是，它会output一些scalar，告诉你说，它要把这个 image 的里面的哪些位置做旋转、缩放，然后，再丢到 CNN 里面，这样你其实会得到比较好的 performance

    Colorful image：如果不是黑白的图而是彩色的图，那么 filter 就不是一个 matrix，是一个 3 层的 立方体（RGB），做完卷积后也是 3 层的

    Convolution vs Fully connected¶

    convolution 其实也是一个 neural network，filter 可以看成 fully connected 的 layer 把一些weight 拿掉，每一个 neuron 只检测 img 中的部分区域，而且公用同一个 weight，所以减小了参数

    CNN 的本质是减小了参数

    Max Pooling¶

    Max Pooling 就是做 subsampling，filter 后得到的 matrix 中以 4 个为一组取最大值，这样 img 就会缩小

    如何微分后面会讲

    Convolution + Max Pooling¶

    Convolution + Max Pooling 可以得到一个较小的 img，可以反复做得到一个 更小的 img

    有一个问题是，一个 Convolution 有 25 个 filter 得到 25 个 feature map，repeat 第二个 Convolution 也有 25 个 filter，是不是会有 25*25 个 feature map

    其实不是，第二次 feature map 会考虑深度，ouyput 还是 25 个

    Flatten¶

    convolution 和 max pooling 之后，就是 FLatten 和 Fully connected Feedforward network 的部分

    Flatten 就是把 feature map 拉直，然后丢进 Fully connected Feedforward network

    CNN in Keras¶

    与 DNN 没什么不同，需要改变的是 network structure，以及 input 的 format

    在 input 中，如果是一个彩图，就需要输入一个高维的 matrix（RGB），叫做 tensor

    model2.add( Convolution2D(25,3,3, input_shape=(28,28,1)) )

    Convolution2D：分别表示 25 个 filter，每个 filter 都是 3*3

    input_shape：以手写数字辨识为例是黑白的，所以是（28，28，1），如果是彩色的就要 (28，28，3)

    model2.add( MaxPooling2D(2,2) )

    MaxPooling2D：分成 2*2 的区域

    以下图为例，第一次 Convolution + Max Pooling 后得到了 25*13*13 的 feature map，第二次的 Convolution 的 filter 其实是 3*3*25 的

    那这样一个 25*13*13 和 25*3*3 的 cubic 做内积之后不应该是 25*11*11 的 cubic 吗？其实还会对每一层做求和运算，这样就是 11*11 了，所以第二次的 Convolution 后 每一个 filter 都得到一个 11*11 的 matrix

    Appendix：CNN in Keras¶

    import numpy as np
from keras.models import Sequential
from keras.layers import Convolution2D, MaxPooling2D, Flatten, Conv2D
from keras.layers.core import Dense, Dropout, Activation
from keras.optimizers import SGD, Adam
from keras.utils import np_utils
from keras.datasets import mnist

# categorical_crossentropy


def load_mnist_data(number):
    # the data, shuffled and  split between train and test sets
    (x_train, y_train), (x_test, y_test) = mnist.load_data()
    x_train = x_train[0:number]
    y_train = y_train[0:number]
    x_train = x_train.reshape(number, 784)
    x_test = x_test.reshape(10000, 784)
    x_train = x_train.astype('float32')
    x_test = x_test.astype('float32')
    # convert class vectors to binary class matrices
    y_train = np_utils.to_categorical(y_train, 10)
    y_test = np_utils.to_categorical(y_test, 10)
    x_train = x_train / 255
    x_test = x_test / 255

    return (x_train, y_train), (x_test, y_test)


if __name__ == '__main__':
    (x_train, y_train), (x_test, y_test) = load_mnist_data(10000)

    # do DNN
    model = Sequential()
    model.add(Dense(input_dim=28 * 28, units=500, activation='relu'))
    model.add(Dense(units=500, activation='relu'))
    model.add(Dense(units=500, activation='relu'))
    model.add(Dense(units=10, activation='softmax'))
    model.summary()

    model.compile(loss='categorical_crossentropy',
                  optimizer='adam', metrics=['accuracy'])

    model.fit(x_train, y_train, batch_size=100, epochs=20)

    result_train = model.evaluate(x_train, y_train)
    print('\nTrain Acc:\n', result_train[1])

    result_test = model.evaluate(x_test, y_test)
    print('\nTest Acc:\n', result_test[1])

    # do CNN
    x_train = x_train.reshape(x_train.shape[0], 1, 28, 28)
    x_test = x_test.reshape(x_test.shape[0], 1, 28, 28)

    model2 = Sequential()
    model2.add(Conv2D(25, (3, 3), input_shape=(
        1, 28, 28), data_format='channels_first'))
    model2.add(MaxPooling2D((2, 2)))
    model2.add(Conv2D(50, (3, 3)))
    model2.add(MaxPooling2D((2, 2)))
    model2.add(Flatten())
    model2.add(Dense(units=100, activation='relu'))
    model2.add(Dense(units=10, activation='softmax'))
    model2.summary()

    model2.compile(loss='categorical_crossentropy',
                   optimizer='adam', metrics=['accuracy'])

    model2.fit(x_train, y_train, batch_size=100, epochs=20)

    result_train = model2.evaluate(x_train, y_train)
    print('\nTrain CNN Acc:\n', result_train[1])
    result_test = model2.evaluate(x_test, y_test)
    print('\nTest CNN Acc:\n', result_test[1])

    What does CNN learn？¶

    what does filter do¶

    在 CNN 中第一个 Convolution 的 filter 是比较容易理解的，看它最终的值是什么就知道它在 detect 什么东西，但是后面的就不好分析了，它考虑的范围不仅是 3*3 的 pixel，还是长宽为 3*3，高为 25 的 cubic（以上面为例子），那一个 filter 到底在做什么呢？

    把第二个 Convolution 的某一个 filter 的 output 拿出来，是一个 11*11 的 matrix，里面的每一个 element 记作 \(a^k_{ij}\)​，k 表示第 k 个 filter

    \(a^k\)​ 叫做 Degree of the activation of the k-th filter，表示现在的第 k 个 filter，它有多被 activate，直观来讲就是描述现在 input 的东西跟第 k 个 filter 有多接近，它对 filter 的激活程度有多少，是 11*11 的 matrix 的 summation

    \[ a^k=\sum\limits^{11}_{i=1}\sum\limits^{11}_{j=1} a^k_{ij} \]

    然后找到一个 img \(x^*\)​，使得 \(a^k\) 最大，那这个 \(x^*\) 就是 filter 在找的图形

    \[ x^*=\arg \max\limits_x a^k \]

    上图挑选了 12 个使得其中 filter 的 activation 最大的图像的例子，都是一些 texture

    what does neuron do¶

    要知道 Flatten 后的 network 里面的 neuron 学到的是什么也是一样的，定义第 j 个 neuron

    的 output 是 \(a_j\)​​，那么下面的 \(x^*\) 就是它学到的图像 $$ x^*=\arg \max\limits_x a_j $$ 相比于 filter 的图像不再是 texture，而是完整的图像

    More Application——Playing Go¶

    Why CNN for Playing Go¶

    AlphaGo 用了 CNN，那什么时候才适合用 CNN 呢？

    要有image该有的那些特性，也就开头所说的，根据观察到的三个 property，我们才设计出了 CNN 这样的network架构：

    • Some patterns are much smaller than the whole image
    • The same patterns appear in different regions
    • Subsampling the pixels will not change the object

    CNN 能够应用在 Alpha-Go 上，是因为围棋有一些特性和图像处理是很相似的

    property 1：围棋中一些 pattern 比如 三个子围一个子

    property 2：同一个 pattern 可以出现在不同位置

    但是其实 AlphaGo 并没有 MaxPooling，因为这显然无法在围棋中使用，所以 structure 需要具体问题，具体分析

    More Application——Speech、Text¶

    Convolution 中的 silde window 不是一定要既有上下也有左右的，比如语音辨识中，通常只在频率方向上移动 filter，而不在时间上；而在文字识别中，filter 只在时间（word）的顺序上移动

    Conclusion¶

    三个 property¶

    • Some patterns are much smaller than the whole image
    • The same patterns appear in different regions
    • Subsampling the pixels will not change the object

    两个架构¶

    convolution 架构：针对 property 1 和 property 2

    max pooling 架构：针对 property 3

    一个理念¶

    针对不同的 application 要设计符合它特性的 network structure，而不是生硬套用，这就是 CNN 架构的设计理念
