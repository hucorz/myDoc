    Logistic Regression¶

    Review¶

    在 Classification 中得出的结果：

    \[ \begin{align} P_{w,b}(C_1|x)=\sigma(z)=\frac{1}{1+e^{-z}} \\ z=w\cdot x+b=\sum\limits_i w_ix_i+b \end{align} \]

    其中 \(w\) 和 \(x\) 都是向量，这个 model 是受 \(w\) 和 \(b\) 控制的，因此我们不必要再去像前面一样计算一大堆东西，而是用这个全新的由 \(w\) 和 \(b\) 决定的 model ——Logistic Regression(逻辑回归)

    Three Steps of machine learning¶

    Step 1：function set¶

    这里的 function set 就是 Logistic Regression——逻辑回归

    \(x_i\)：input，\(w_i\)：weight，\(b\)：bias，\(\sigma(z)\)：sigmoid function

    Step 2：Goodness of a function¶

    设计一个 Loss Function

    知道了某一组 \(w\) 和 \(b\) 就可以计算出 Posterior Probability（后验概率），然后利用后验概率可以计算出极大似然估计，那么能够使极大似然估计值最大的那组 \(w^*\) 和 \(b^*\) 就是最有可能产生当前 Training Data 的一组参数

    这里假定二元分类，class1 的概率是 1-class2 的概率

    将上式变形：

    \[ w^*,b^*=\arg \max\limits_{w,b} L(w,b)=\arg\min\limits_{w,b}(-\ln L(w,b)) \]
    \[ \begin{align} -\ln L(w,b)=&-\ln f_{w,b}(x^1)\\ &-\ln f_{w,b}(x^2)\\ &-\ln(1-f_{w,b}(x^3))\\ &-... \end{align} \]

    为了统一格式，把 output \(\hat{y}=1\)​ 代表 class1，output \(\hat{y}=0\)​​ 代表 class2

    \[ \begin{align} -\ln L(w,b)=&-[\hat{y}^1 \ln f_{w,b}(x^1)+(1-\hat{y}^1)ln(1-f_{w,b}(x^1))]\\ &-[\hat{y}^2 \ln f_{w,b}(x^2)+(1-\hat{y}^2)ln(1-f_{w,b}(x^2))]\\ &-[\hat{y}^3 \ln f_{w,b}(x^3)+(1-\hat{y}^3)ln(1-f_{w,b}(x^3))]\\ &\ -... \end{align} \]
    \[ -\ln L(w,b)=\sum\limits_n -[\hat{y}^n \ln f_{w,b}(x^n)+(1-\hat{y}^n) \ln(1-f_{w,b}(x^n))] \]

    这个形式其实就是 两个 Bernoulli distribution (两点分布)的 cross entropy (交叉熵)，把这个作为 Loss Function

    于是我们把 极大似然估计 的式子变成了 交叉熵 作为 Loss Function 以方便微分

    Cross Entropy¶

    笔记 | 什么是Cross Entropy - 知乎

    Cross Entropy: 一般用来量化两个概率分布之间差异的损失函数

    假设对于一个数据的真实分布为 p，预测分布为 q

    那么 \(H(p,q)=-\sum_{x}{p(x)\ log\ q(x)}\) 其中 x 是类别数

    Cross Entropy 交叉熵的含义是表达两个 distribution 有多接近，如果 p 和 q 这两个 distribution 一模一样的话，那它们算出来的 Cross Entropy 就是 0 (详细解释在“信息论”中)，而这里\(f(x^n)\)表示 function 的 output，\(\hat{y}^n\) 表示预期的 target，因此交叉熵实际上表达的是希望这个 function 的 output和它的 target 越接近越好

    negative log loss（负对数似然估计）是 Cross Entropy 的一个应用，在单样本的情况下假设真实分布为 p，表示一个向量，其中 \(p_k=1\)，其余为 0，那么 \(H(p, q) =-log(q_k)\) ，也就是说，交叉熵只关心对正确类别的预测概率，希望其尽量大

    step 3：Find the best function¶

    即找到使 Loss Function 即 Cross Entropy 最小的一组 \(w^*,b^*\)，这里运用 Gradient Descent

    \[ \begin{align} P_{w,b}(C_1|x)=\sigma(z)=\frac{1}{1+e^{-z}} \\ z=w\cdot x+b=\sum\limits_i w_ix_i+b \end{align} \]
    \[ \begin{align} f_{w,b}(x)=P_{w,b}(C_1|x) \end{align} \]
    \[ -\ln L(w,b)=\sum\limits_n -[\hat{y}^n \ln f_{w,b}(x^n)+(1-\hat{y}^n) \ln(1-f_{w,b}(x^n))] \]

    Sigmoid Function 的微分公式： $$ \frac{\partial \sigma(z)}{\partial z}=\sigma(z)(1-\sigma(z)) $$

    计算 \(-\ln L(w,b)=\sum\limits_n -[\hat{y}^n \ln f_{w,b}(x^n)+(1-\hat{y}^n) \ln(1-f_{w,b}(x^n))]\) 对 \(w_i\) 的偏微分，其中 \(\hat{y}^n\) 和 \(1-\hat{y}^n\) 是常数，只需要分别求出 \(\ln f_{w,b}(x^n)\) 和 \(\ln (1-f_{w,b}(x^n))\) 对 \(w_i\) 的偏微分即可，整体推导过程如下：

    结果带入得：

    所以： $$ w_{i+1}=w_i-\eta \sum\limits_{n}-(\hat{y}^n-f_{w,b}(x^n))x_i^n $$ \(-\eta\)：learning rate

    \(x_i\)：data

    \(\hat{y}^n-f_{w,b}(x^n)\)：与理想 function set 的差距，差距越大 update 的步伐越大

    同理可得： $$ b_{i+1}=w_i-\eta \sum\limits_{n}-(\hat{y}^n-f_{w,b}(x^n)) $$ 我们会发现 Logistic Regression 的结果是和 Linear Regresison 的式子是一样的

    Comparison¶

    Logistic Regression && Linear Regression¶

    • 两者都是通过对每一个 feature \(x_i\) 加权求和后加上一个常数，区别在于 Logistic Regression 最后又通过了一个 Sigmoid Function，它的结果是 0~1，而 Linear Regression 不经过 Sigmoid Function，结果可以是任意值
    • Logistic Regression 的 Loss Function 用的交叉熵定义，而 Linear Regression 用的 Square Error

    Logistic Regression + Square Error ？¶

    假设使用 Logistic Regression + Square Error， 如果此时 \(\hat{y}^n=1\)，而且 function 的 output \(f_{w,b}(x^n)=1\) 的话，说明现在离 target 很接近了，\(f_{w,b}(x)-\hat{y}=0\)，于是得到的微分 \(\frac{\partial L}{\partial w_i}\) 等于 0，这件事情是很合理的；但是当 function 的 output \(f_{w,b}(x^n)=0\)的时候，说明离 target 还很遥远，但是由于在step3 中求出来的 update 表达式中有一个\(f_{w,b}(x^n)\)，因此这个时候也会导致得到的微分 \(\frac{\partial L}{\partial w_i}\) 也等于 0

    Discriminative && Generative¶

    把 Logistic Regression 的方法称之为 Discriminative 的方法，用 Gaussian 来描述 Posterior Probability 的方法称之为 Generative 的方法

    他们的 Function Set 都是一样的，都是 \(P(C_1|x)=\sigma(w\cdot x+b)\)

    Logistic Regression 是通过 Gradient Descent 的方法直接找出 \(b\) 和 \(w\)，而 Generative Model 的方法是先算 \(u_1,u_2,\Sigma^{-1}\)，再求 \(b\) 和 \(w\)

    但这两者方法的出来的 \(b\) 和 \(w\) 是不一样的

    原因是在 Logistic Regression 中没有做任何实质性的假设，单纯的去找 \(w\) 和 \(b\)，而在 Generative Model 中假设了 Gaussion Distribution

    Discriminative 的方法常常会比 Generative 的方法表现得更好

    discriminative model 并不是在所有的情况下都可以赢过 Generative model，discriminative model 是十分依赖于 data 的，当 data 数量不足或是 data 本身的 label就有一些问题，那Generative model做一些假设，反而可以把 data 的不足或是有问题部分的影响给降到最低

    在 Generative model 中，priors probabilities 和 class-dependent probabilities 是可以拆开来考虑的，以语音辨识为例，现在用的都是 neural network，是一个discriminative 的方法，但事实上整个语音辨识的系统是一个 Generative 的 system，它的 prior probability 是某一句话被说出来的几率，而想要 estimate 某一句话被说出来的几率并不需要有声音的 data，可以去互联网上爬取大量文字，就可以计算出某一段文字出现的几率，并不需要声音的 data，这个就是 language model，而 class-dependent 的部分才需要声音和文字的配合，这样的处理可以把 prior 预测地更精确

    Conclusion¶

    对于分类问题（主要是二元分类），一般有两种方法处理：Generative 的方法，Discriminative 的方法

    \[ \begin{align} P(C_i|x)=\frac{P(C_i)P(x|C_i)}{\sum\limits_{j=1}^nP(C_j)P(x|C_j)} \\ \sigma(z)=\frac{1}{1+e^{-z}}=\frac{1}{1+e^{-(b+\sum\limits_k w_k x_k)}} \end{align} \]

    两者的区别在于：

    Generative model 通常会假设一个概率分布，然后去利用极大似然估计计算 \(b\) 和 \(w\)；Discriminative model 不做任何假设，利用交叉熵和 gradient descent 计算 \(b\) 和 \(w\)

    Generative model 的好处是，它对data的依赖并没有像discriminative model 那么严重，在data数量少或者data本身就存在noise的情况下受到的影响会更小，而它还可以做到 Prior 部分与 class-dependent 部分分开处理，如果可以借助其他方式提高 Prior model 的准确率，对整一个 model 是有所帮助的 (比如前面提到的语音辨识)

    而 Discriminative model 的好处是，在 data 充足的情况下，它训练出来的 model 的准确率一般是比 Generative model 要来的高的

    Multi-class Classification¶

    Softmax¶

    假设有三个 Class：\(C_1,C_2,C_3\)，每一个 Class 都有自己的 weight 和 bias，分别是 \(w_1,w_2,w_3\) (vector)，\(b_1,b_2,b_3\) (const)，input x 也是一个 vector

    那么 \(z_i=w^ix+b_i\)

    softmax：强化最大值，因为做 exponential 会让原本大的值变得更大

    然后把 \(z_1,z_2,z_3\)丢进一个 softmax 的 function

    • 取 exponential，得到 \(e^{z_1},e^{z_2},e^{z_3}\)
    • 归一化，得到 \(y_1=\frac{e^{z_1}}{\sum\limits_{j=1}^3 e^{z_j}}\)、\(y_2=\frac{e^{z_2}}{\sum\limits_{j=1}^3 e^{z_j}}\)、\(y_3=\frac{e^{z_3}}{\sum\limits_{j=1}^3 e^{z_j}}\)

    softmax 的 output，就拿来当 z 的 probability

    multi-class classification 的过程¶

    input x 经过三个式子后分别生成 \(z_1,z_2,z_3\)，经过 softmax 后转换成 \(y_1,y_2,y_3\)，这分别是三个 class 的 posterior probability

    为了计算交叉熵，不能把 targrt 设为 1,2,3，这里使用向量

    \[ \hat{y}= \begin{bmatrix} 1\\0\\0 \end{bmatrix}_{x \ ∈ \ class 1} \hat{y}= \begin{bmatrix} 0\\1\\0 \end{bmatrix}_{x \ ∈ \ class 2} \hat{y}= \begin{bmatrix} 0\\0\\1 \end{bmatrix}_{x \ ∈ \ class 3} \]

    然后计算 \(y，\hat{y}\) 之间的交叉熵，\(-\sum\limits_{i=1}^3 \hat{y}_i \ln y_i\)

    Limitation of Logistic Regression¶

    Logistic Regression 有很强的限制，因为他的 boundary 是一条直线，而对于下面的例子

    \(x_1\) \(x_2\) label
    0 0 class2
    0 1 class1
    1 0 class1
    1 1 class2

    在坐标轴上是不可能画一条 boundary 来区分 class1 和 class2 的

    Feature Transformation¶

    如果原来的 feature 不好做划分，可以用 Feature Transformation，转换之后变得可以划分。

    假定新的 x 轴 \(x'_1\) 表示原来的点到 \(\begin{bmatrix}0\\0\end{bmatrix}\) 的距离，新的 y 轴 \(x'_2\) 表示的是原来的点到 \(\begin{bmatrix}1\\1\end{bmatrix}\) 的距离，重新映射之后就变得可划分了

    麻烦的是很多时候我们并不知道怎么做 Feature Transformation，于是我们会希望这个 Transformation 是机器自己产生的，怎么让机器自己产生呢？我们可以让很多Logistic Regression cascade（连接）起来

    即先通过一个 Logistic Regression 的 Transform 使得新的 feature \(x'_1,x'_2\) 是可以划分的，然后再经过一个 Logistic Regression 就可以把 class1 和 class2 分开

    所以整个流程是：先通过 n（n为每个样本点的feature数量）个 Logistic Regression 做 feature Transformation，然后再用一个 Logistic Regression 做 classification（如果是多元分类需要用到多个 Logistic Regression 画出多条直接划分所有的类）

    我们如果把这其中的每个 Logistic Regression 叫做一个 Neuron，把这些 Logistic Regression 串起来形成的 network 叫做 Neural Network，十九类神经网络，就是 Deep Learning
