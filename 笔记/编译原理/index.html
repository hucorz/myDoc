    编译原理

    参考：自己动手写编译器 — 自己动手写编译器 (pandolia.net)

    Flex¶

    例子：

    %{
#define T_WORD 1
int numChars = 0, numWords = 0, numLines = 0;
%}

WORD        ([^ \t\n\r\a]+)

%%
\n          { numLines++; numChars++; }
{WORD}      { numWords++; numChars += yyleng; return T_WORD; }
<<EOF>>     { return 0; }
.           { numChars++; }
%%

int main() {
    int token_type;
    while (token_type = yylex()) {
        printf("WORD:\t%s\n", yytext);
    }
    printf("\nChars\tWords\tLines\n");
    printf("%d\t%d\t%d\n", numChars, numWords, numLines);
    return 0;
}

int yywrap() {
    return 1;
}

    规则说明：

    %{
Declarations
%}
Definitions
%%
Rules
%%
User subroutines

    Declarations，声明，都是 C 代码，这些代码会被原样的复制到 lex.yy.c 文件中，一般在这里声明一些全局变量和函数，这样在后面可以使用这些变量和函数

    Definitions，定义，在这里可以定义正则表达式中的一些名字，可以在 规则（Rules） 段被使用

    Rules 部分会被翻译为一个 yylex 函数，此函数扫描到一个可以和 Rules 中匹配的字符串时会执行后面的代码；有多条规则匹配时选择最长匹配，匹配长度相同时选择最前面的规则

    User subroutines 部分的代码会原样复制到 lex.yy.c 代码的最后

    输入文件中最后一行的 yywrap 函数的作用是将多个输入文件打包成一个输入，当 yylex 函数读入到一个文件结束（EOF）时，它会向 yywrap 函数询问， yywrap 函数返回 1 的意思是告诉 yylex 函数后面没有其他输入文件了，此时 yylex 函数结束，yywrap 函数也可以打开下一个输入文件，再向 yylex 函数返回 0 ，告诉它后面还有别的输入文件，此时 yylex 函数会继续解析下一个输入文件；如果只需要处理一个文件，默认返回 1 即可

    宏：

    ECHO            //将匹配字符串原样输出

    内置变量:

    yytext         //匹配成功的字符串,可以直接当字符串用
yylval         //flex进行scanner，将数值存入yylval,而bison可以读取yylval之中的值，此变量类型为 YYSTYPE

    可以通过修改 YYSTYPE 来修改 yylval 的类型，且 YYSTYPE 定义必须在 #include"xxx.tab.h" 之前，如果需要 yylval 支持多种类型，可以使用 %union 声明，此声明会修改 yylval 的类型，如：

    %union
{
    int intValue;
    float floatValue;
}
// 那么在使用 yylval 时就需要写
[a-zA-Z][a-zA-Z0-9]* {
 yylval.stringValue = strdup(yytext);
 return IDENTIFIER;
}

    Bison¶

    %{
#include <stdio.h>
void yyerror(const char* msg) {}
%}

%token T_NUM

%left '+' '-'
%left '*' '/'

%%

S   :   S E '\n'        { printf("ans = %d\n", $2); }
    |   /* empty */     { /* empty */ }
    ;

E   :   E '+' E         { $$ = $1 + $3; }
    |   E '-' E         { $$ = $1 - $3; }
    |   E '*' E         { $$ = $1 * $3; }
    |   E '/' E         { $$ = $1 / $3; }
    |   T_NUM           { $$ = $1; }
    |   '(' E ')'       { $$ = $2; }
    ;

%%

int main() {
    return yyparse();
}

    规则说明：

    %{
Declarations
%}
Definitions
%%
Productions
%%
User subroutines

    Declarations && User subroutines 和 flex 类似，这2部分的代码会原样复制到生成的 c 代码中

    Definitions 中定义一些 bison 专有的变量

    Productions 中写语法产生式，用 : 来表示 -> ，| 隔开同一个终结符的不同的产生式， ; 表示此产生式的结束，产生式右边是 ε 时，不需要写任何符号，一般用一个注释 / empty / 代替，此部分会被翻译为一个 yyparse 函数

    bison 会将 Productions 段里的第一个产生式的左边的非终结符当作语法的起始符号，同时，为了保证起始符号不位于任何产生式的右边， bison 会自动添加一个符号（如 S’ ）以及一条产生式（如 S’ -> S ），而将这个新增的符号当作解析的起始符号

    Bison 会自动识别非终结符；终结符中，单字符 token 用单引号括起来（如 '\n' '+'），其他 token 需要在 Definition 中定义好，如例子中的 token T_NUM， bison 会自动为这种 token 分配一个编号，再写到 y.tab.h 文件中去，可以在 flex 中直接使用这个文件

    可以在 Definitions 段定义符号的优先级

    %left '+' '-'
%left '*' '/'

    left 表示 左结合，下面行的符号的优先级高于上面行的符号，同一行的符号的优先级相同

    bison 生成一个 C 语言的 LALR(1) 动作表，并输出到 y.tab.c 文件中去，另外，还会将这个动作表以及语法中的相关要素以可读的文字形式输出到 y.output 文件中去，此文件中会有每个状态以及状态中所有的形态

    token stream 合法 yyparse 返回 0，不合法则调用 yyerror 并返回 1

    yyparse 函数不仅维持一个状态栈，它还维持一个符号属性栈，当它执行 shift 动作时，它除了将相应的状态压入状态栈之外，还会将一个类型为 YYSTYPE （默认和 int 相同）、名为 yylval 的全局变量的数值压入到属性栈内，而在 reduce 动作时，可以用 $1, $2, ... $n 来引用属性栈的属性， reduce 动作不仅将相应的状态出栈，还会将同样数量的属性出栈，这些属性和 reduce 产生式的右边的符号是一一对应的，同时，用 $$ 代表产生式左边的终结符，在 reduce 动作里可以设置 $$ 的值，当执行 goto 动作时，除了将相应的状态入栈，还会将 $$ 入栈

    如果要使用 $$ $1 这种需要在 \%token 后面规定类型

    例子¶

    Flex 代码

    %{
#define YYSTYPE char *
#include "y.tab.h"
int cur_line = 1;
void yyerror(const char *msg);
void unrecognized_char(char c);
%}

OPERATOR        [-/+*()=;]
INTEGER         [0-9]+
IDENTIFIER      [_a-zA-Z][_a-zA-Z0-9]*
WHITESPACE      [ \t]*

%%
{OPERATOR}      { return yytext[0]; }
{INTEGER}       { yylval = strdup(yytext); return T_IntConstant; }
{IDENTIFIER}    { yylval = strdup(yytext); return T_Identifier; }
{WHITESPACE}    { /* ignore every whitespcace */ }
\n              { cur_line++; }
.               { unrecognized_char(yytext[0]); }
%%

int yywrap(void) { 
    return 1;
}

void unrecognized_char(char c) {
    char buf[32] = "Unrecognized character: ?";
    buf[24] = c;
    yyerror(buf);
}

void yyerror(const char *msg) {
    printf("Error at line %d:\n\t%s\n", cur_line, msg);
    exit(1);
}

    Bison 代码：

    %{
#include <stdio.h>
#include <stdlib.h>
void yyerror(const char*);
#define YYSTYPE char *
%}

%token T_IntConstant T_Identifier

%left '+' '-'
%left '*' '/'
%right U_neg

%%

S   :   Stmt
    |   S Stmt
    ;

Stmt:   T_Identifier '=' E ';'  { printf("pop %s\n\n", $1); }
    ;

E   :   E '+' E                 { printf("add\n"); }
    |   E '-' E                 { printf("sub\n"); }
    |   E '*' E                 { printf("mul\n"); }
    |   E '/' E                 { printf("div\n"); }
    |   '-' E %prec U_neg       { printf("neg\n"); }
    |   T_IntConstant           { printf("push %s\n", $1); }
    |   T_Identifier            { printf("push %s\n", $1); }
    |   '(' E ')'               { /* empty */ }
    ;

%%

int main() {
    return yyparse();
}

    U_neg 表示取反，右结合，%prec U_neg 表示次产生式的优先级等于 U_neg，否则次产生式的优先级等于产生式中最右边的、定义了优先级的符号的优先级
