    二、基础篇¶

    2.1 TCP/IP 网络模型¶

    为什么要有 TCP/IP 网络模型？：同一个设备之间的通信方式有很多，但是不同设备多种多样，需要统一的协议

    TCP/IP 网络模型：

    • 应用层（Application Layer）：HTTP，FTP，DNS，STMP
    • 传输层（Transport Layer）：TCP，UDP
    • 网络层（Internet Layer）：IP
    • 网络接口层（Link Layer）：MAC

    2.2 键入网址到网页显示，期间发生了什么？¶

    • 解析 URI：
    • URI 组成
    • HTTP 请求：请求行，消息头，消息体
    • DNS
    • 域名的层级关系
    • 域名的解析流程
    • 协议栈：通过 DNS 获取到 IP 后，就可以把 HTTP 的传输工作交给操作系统中的协议栈
    • TCP
    • TCP 报文头格式：源端口号 目标端口号，确认号，状态位，窗口大小（流量控制，拥塞控制）
    • 三次握手
    • MTU，MSS
    • IP
    • IP 包头：源地址IP，目标地址 IP
    • 路由表
    • MAC：以太网中
    • MAC 头部
    • ARP
    • 网卡：数字信号 -> 电信号
    • 开头加上报头和起始帧分界符，在末尾加上用于检测错误的帧校验序列
    • 交换机：交换机端口不具有 MAC 地址
    • MAC 地址表，找不到就广播
    • 路由器：路由器的各个端口都具有 MAC 地址和 IP 地址
    • 路由表：查表时匹配主机号，拿目标地址和表中每项的子网掩码&后再和对应的项的目标地址匹配
    • 如果路由表中的网关为空，说明 接收方 IP 地址就是终点
    • 包传输过程中的 源 IP 地址和 目标 IP 地址不会变，变化的是 MAC 地址

    2.3 Linux 系统是如何收发网络包的？¶

    OSI 网络模型（Open System Interconnection Reference Model）：

    • 应用层，负责给应用程序提供统一的接口；
    • 表示层，负责把数据转换成兼容另一个系统能识别的格式；
    • 会话层，负责建立、管理和终止表示层实体之间的通信会话；
    • 传输层，负责端到端的数据传输；
    • 网络层，负责数据的路由、转发、分片；
    • 数据链路层，负责数据的封帧和差错检测，以及 MAC 寻址；
    • 物理层，负责在物理网络中传输数据帧；

    TCP/IP 网络模型：

    • 应用层，负责向用户提供一组应用程序，比如 HTTP、DNS、FTP 等;
    • 传输层，负责端到端的通信，比如 TCP、UDP 等；
    • 网络层，负责网络包的封装、分片、路由、转发，比如 IP、ICMP 等；
    • 网络接口层，负责网络包在物理网络中的传输，比如网络包的封帧、 MAC 寻址、差错检测，以及通过网卡传输网络帧等；
